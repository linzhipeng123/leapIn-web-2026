# 语言切换测试指南

## 测试步骤

### 1. 启动开发服务器

```bash
pnpm dev
```

### 2. 访问法律页面

在浏览器中打开以下任一页面：
- http://localhost:4321/legal/privacy
- http://localhost:4321/legal/terms
- http://localhost:4321/legal/security

### 3. 测试语言切换

1. **查看默认语言**
   - 首次访问时，页面应显示中文内容（默认语言）
   - 标题和内容都应该是中文

2. **切换到英文**
   - 点击页面右上角的语言切换器（显示 "Cn"）
   - 在下拉菜单中选择 "En"
   - 页面会刷新
   - 刷新后，标题和内容应该变为英文

3. **切换回中文**
   - 点击页面右上角的语言切换器（现在显示 "En"）
   - 在下拉菜单中选择 "Cn"
   - 页面会刷新
   - 刷新后，标题和内容应该变回中文

4. **测试语言持久化**
   - 切换到英文后，刷新页面
   - 页面应该保持英文显示
   - 打开新标签页访问同一页面，也应该显示英文
   - 这证明语言设置已保存在 localStorage 中

### 4. 验证内容

检查以下内容是否正确显示：

#### 隐私政策页面 (/legal/privacy)
- **中文标题**：隐私政策
- **英文标题**：Privacy Policy
- 内容应包含多个章节（关于本隐私政策、隐私政策范围等）

#### 服务条款页面 (/legal/terms)
- **中文标题**：服务条款
- **英文标题**：Terms of Service
- 内容应包含多个章节（接受条款、服务描述等）

#### 安全政策页面 (/legal/security)
- **中文标题**：安全政策
- **英文标题**：Security Policy
- 内容应包含多个章节（安全承诺、数据安全等）

### 5. 测试样式

确认以下样式正确应用：

- [ ] 标题居中显示
- [ ] 标题字体大小合适（2.5rem）
- [ ] 内容最大宽度为 900px 并居中
- [ ] 标题层级清晰（h1-h5）
- [ ] 列表格式正确（有序和无序）
- [ ] 链接颜色为蓝色（#5386FA）
- [ ] 邮箱高亮样式生效（渐变背景）
- [ ] 移动端响应式布局正常

### 6. 测试浏览器兼容性

在以下浏览器中测试：
- [ ] Chrome/Edge
- [ ] Firefox
- [ ] Safari

### 7. 测试移动端

使用浏览器的开发者工具切换到移动设备视图：
- [ ] iPhone 12/13/14
- [ ] iPad
- [ ] Android 手机

确认：
- [ ] 标题字体大小调整为 1.5rem
- [ ] 内容区域 padding 调整为 2rem 1rem
- [ ] 语言切换器可正常使用

## 调试技巧

### 查看 localStorage

在浏览器控制台中运行：
```javascript
// 查看当前语言设置
console.log(localStorage.getItem('language'));

// 手动设置语言
localStorage.setItem('language', 'en'); // 或 'zh'
location.reload(); // 刷新页面查看效果
```

### 查看元素属性

在浏览器开发者工具中：
1. 选择标题或内容元素
2. 查看 `data-lang-zh` 和 `data-lang-en` 属性
3. 确认两个属性都包含正确的内容

### 查看控制台错误

打开浏览器控制台（F12），检查是否有：
- JavaScript 错误
- 文件加载失败
- Markdown 解析错误

## 常见问题排查

### 问题 1：语言切换后内容没有变化

**可能原因：**
- Markdown 文件未正确读取
- 文件名不匹配

**解决方法：**
1. 检查 `src/content/legal/` 目录中是否存在对应的文件
2. 确认文件名格式：`{page-name}-{lang}.md`
3. 查看浏览器控制台是否有错误信息

### 问题 2：页面显示 "内容不可用" 或 "Content not available"

**可能原因：**
- Markdown 文件不存在或路径错误
- 文件读取权限问题

**解决方法：**
1. 确认文件存在：`ls -la src/content/legal/`
2. 检查文件权限：`chmod 644 src/content/legal/*.md`
3. 重启开发服务器

### 问题 3：样式显示不正常

**可能原因：**
- CSS 未正确加载
- 全局样式冲突

**解决方法：**
1. 清除浏览器缓存
2. 硬刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）
3. 检查浏览器开发者工具中的 Styles 面板

### 问题 4：移动端显示异常

**可能原因：**
- 媒体查询未生效
- viewport 设置问题

**解决方法：**
1. 检查 Layout.astro 中的 viewport meta 标签
2. 确认媒体查询断点（768px）
3. 使用浏览器开发者工具的响应式设计模式测试
